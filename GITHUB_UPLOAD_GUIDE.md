# GitHub Upload Guide

## Recommended Repository Name

```text
salzburg-urban-amenities-webgis
```

## Repository Description

```text
Professional Leaflet WebGIS application for exploring public amenities in Salzburg, Austria.
```

## What to Upload

Upload the full contents of the project folder:

```text
index.html
README.md
portfolio_project_text.md
GITHUB_UPLOAD_GUIDE.md
css/
js/
data/
docs/
```

Do not upload only the ZIP file. GitHub should show the actual folders and files.

## Running with GitHub Pages

This project is a static HTML/CSS/JavaScript WebGIS application, so it can be hosted directly on GitHub Pages.

After uploading the project:

1. Go to the repository on GitHub.
2. Open **Settings**.
3. Click **Pages**.
4. Under **Build and deployment**, choose **Deploy from a branch**.
5. Select the `main` branch and `/root` folder.
6. Click **Save**.

The live application URL will usually look like:

```text
https://YOUR-GITHUB-USERNAME.github.io/salzburg-urban-amenities-webgis/
```

## Adding It to Your ePortfolio

On your main ePortfolio website, add a project card with:

**Title:** Explore Salzburg: Interactive WebGIS Application for Urban Amenities  
**Category:** Geovisualization and Advanced Cartography  
**Tools:** Leaflet, JavaScript, HTML, CSS, GeoJSON  
**Repository link:** your GitHub repository URL  
**Live app link:** your GitHub Pages URL
